# GSA (Apple's Authentication Service) API Documentation

## Overview

The GSA (Apple's Authentication Service) is Apple's authentication system used for signing into Apple services. AltSign uses GSA to authenticate users with their Apple ID and handle two-factor authentication (2FA).

## Base URL
```
https://gsa.apple.com
```

## Authentication Protocol

GSA uses the SRP (Secure Remote Password) protocol, defined in [RFC 5054](https://tools.ietf.org/html/rfc5054), to authenticate users without sending the password over the network.

### SRP Protocol Flow

1. **Client → Server**: Client sends public key (A) and username
2. **Server → Client**: Server sends public key (B), salt (s), and iteration count (i)
3. **Client → Server**: Client sends verification message (M1)
4. **Server → Client**: Server sends verification message (M2) to prove authenticity
5. **Session Key**: Both parties derive a shared session key

### Key Components

- **A (Client Public Key)**: Generated by client using SRP
- **B (Server Public Key)**: Generated by server using SRP
- **s (Salt)**: Random salt used for password derivation
- **i (Iterations)**: Number of PBKDF2 iterations for password derivation
- **x (Derived Password Key)**: Derived from password, salt, and iterations
- **M1 (Client Verification Message)**: Proves client knows password
- **M2 (Server Verification Message)**: Proves server knows password verifier
- **K (Session Key)**: Shared secret for encrypted communication

## Anisette Data

Anisette data is device-specific authentication data required for all GSA requests. It simulates a trusted Apple device.

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `machineID` | String | Device machine identifier |
| `oneTimePassword` | String | One-time password for authentication |
| `localUserID` | String | Local user identifier |
| `routingInfo` | UInt64 | Routing information |
| `deviceUniqueIdentifier` | String | Unique device identifier (UDID) |
| `deviceSerialNumber` | String | Device serial number |
| `deviceDescription` | String | Device description (e.g., "iPhone12,1") |
| `date` | Date | Current date/time |
| `locale` | Locale | Device locale (e.g., "en_US") |
| `timeZone` | TimeZone | Device timezone (e.g., "America/Los_Angeles") |

### Obtaining Anisette Data

Anisette data can be obtained from:
- A jailbroken iOS device running anisette server
- A macOS machine with iCloud credentials
- Third-party anisette services

## Endpoints

### 1. Authentication Handshake (Init)

Initiates the SRP authentication handshake.

**Endpoint:**
```
POST https://gsa.apple.com/grandslam/GsService2
```

**Operation:** `init`

**Request Headers:**
```http
Content-Type: text/x-xml-plist
X-MMe-Client-Info: <device description>
Accept: */*
User-Agent: akd/1.0 CFNetwork/978.0.7 Darwin/18.7.0
```

**Request Body (Property List):**
```xml

        <data><base64 encoded client public key A></data>
```

**Response Fields:**
- `c`: Cookie value for subsequent requests
- `s`: Salt for password derivation
- `i`: Number of PBKDF2 iterations
- `B`: Server's public key
- `sp`: Password scheme (s2k or s2k_fo for hexadecimal)

### 2. Authentication Handshake (Complete)

Completes the SRP authentication by sending the client verification message.

**Endpoint:**
```
POST https://gsa.apple.com/grandslam/GsService2
```

**Operation:** `complete`

**Request Headers:** Same as init

**Request Body (Property List):**
```xml

<plist version="1.0">
<dict>
    <key>Header</key>
    <dict>
        <key>Version</key>
        <string>1.0.1</string>
    </dict>
    <key>Request</key>
    <dict>
        <key>c</key>
        <string><c value from init></string>
        <key>cpd</key>
        <dict>
            <!-- Same cpd as init -->
        </dict>
        <key>M1</key>
        <data><base64 encoded client verification message></data>
        <key>o</key>
        <string>complete</string>
        <key>u</key>
        <string><lowercase apple id></string>
    </dict>
</dict>
</plist>
```

**Response Body (Property List):**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Response</key>
    <dict>
        <key>M2</key>
        <data><base64 encoded server verification message></data>
        <key>spd</key>
        <data><encrypted data containing dsid and tokens></data>
        <key>Status</key>
        <dict>
            <key>au</key>
            <string><auth type></string>
            <key>ec</key>
            <integer>0</integer>
        </dict>
    </dict>
</dict>
</plist>
```

**Response Fields:**
- `M2`: Server's verification message (must be verified)
- `spd`: Encrypted data containing DSID and authentication tokens
- `Status.au`: Authentication type
  - `trustedDeviceSecondaryAuth`: Trusted device 2FA required
  - `secondaryAuth`: SMS 2FA required
  - (absent): No 2FA required

### 4. Trusted Device 2FA Request

Requests a 2FA code to be sent to a trusted device.

**Endpoint:**
```
POST https://gsa.apple.com/auth/verify/trusteddevice
```

**Request Headers:**
```http
Accept: application/x-buddyml
Accept-Language: en-us
Content-Type: application/x-plist
User-Agent: Xcode
X-Apple-App-Info: com.apple.gs.xcode.auth
X-Xcode-Version: 11.2 (11B41)
X-Apple-Identity-Token: <base64 encoded dsid:idmsToken>
X-Apple-I-MD-M: <machine ID>
X-Apple-I-MD: <one-time password>
X-Apple-I-MD-LU: <local user ID>
X-Apple-I-MD-RINFO: <routing info>
X-Mme-Device-Id: <device unique identifier>
X-MMe-Client-Info: <device description>
X-Apple-I-Client-Time: <ISO 8601 date>
X-Apple-Locale: <locale>
X-Apple-I-TimeZone: <timezone>
```

**Request Body:** Empty

**Response:** Success if code sent successfully

### 5. Trusted Device 2FA Verify

Verifies the 2FA code from a trusted device.

**Endpoint:**
```
POST https://gsa.apple.com/grandslam/GsService2/validate
```

**Request Headers:** Same as request, plus:
```http

```

**Request Body:** Empty

**Response (Property List):**

```xml
<plist version="1.0">
<dict>
    <key>ec</key>
    <integer>0</integer>
</dict>
</plist>
```

## Encryption

GSA uses two encryption methods for secure communication:

### CBC Decryption

Used to decrypt the `spd` field in the complete response.

**Algorithm:** AES-256-CBC with PKCS7 padding

**Key Derivation:** HMAC-SHA256 of session key with "extra data key:"

**IV Derivation:** HMAC-SHA256 of session key with "extra data iv:"

### GCM Decryption

Used to decrypt the `et` field in the apptokens response.

**Algorithm:** AES-256-GCM

**Format:**
- 3 bytes: version
- 16 bytes: initialization vector
- N bytes: ciphertext
- 16 bytes: authentication tag

## Error Codes

| Error Code | Description |
|------------|-------------|
| 0 | Success |
| -20101 | Incorrect credentials |
| -22406 | Incorrect credentials |
| -22421 | Invalid anisette data |
| -21669 | Incorrect verification code (2FA) |

## Implementation Details

### SRP Group

AltSign uses the 2048-bit SRP group from RFC 5054:
```
N (prime): 2048-bit prime
g (generator): 2
```

### Password Derivation

The password is derived using PBKDF2-HMAC-SHA256:
```
x = PBKDF2-HMAC-SHA256(SHA256(password), salt, iterations, 32 bytes)
```

If `sp` is `s2k_fo`, the SHA256 digest is converted to hexadecimal before PBKDF2.

### Checksum Calculation

The checksum for apptokens is calculated using HMAC-SHA256:
```
checksum = HMAC-SHA256(sessionKey, "apptokens" + dsid + appName)
```

```swift
// Create anisette data
let anisetteData = ALTAnisetteData(

// Authenticate

api.authenticate(
    appleID: "your@email.com",
    password: "your-password",
    anisetteData: anisetteData,
    verificationHandler: { completion in
        // Prompt user for 2FA code
        print("Enter 2FA code:")
        if let code = readLine() {
            completion(code)
        } else {
            completion(nil)
        }
    }

    if let error = error {
        print("Authentication failed: \(error)")
    } else {
        print("Authenticated successfully")
        print("Account: \(account?.name ?? "Unknown")")
        print("DSID: \(session?.dsid ?? "Unknown")")
        
        // Use session for Developer Services API
        // ...
    }
}
```

## Security Considerations

1. **Session Security**: The session key and authentication tokens must be kept secure
2. **Anisette Data**: Should be obtained from trusted sources only
3. **Password Security**: Password is never sent over the network (SRP protocol)
4. **2FA**: Always handle 2FA codes securely and don't log them
5. **Token Expiration**: Authentication tokens may expire; handle re-authentication

## Troubleshooting

### Common Issues

1. **Invalid Anisette Data (-22421)**
   - Ensure anisette data is current and valid
   - Check that all required fields are present
   - Verify device description matches a real Apple device

2. **Incorrect Credentials (-20101, -22406)**
   - Verify Apple ID and password are correct
   - Ensure Apple ID is lowercase
   - Check for account lockouts

3. **2FA Required**
   - Always provide a verification handler
   - Handle both SMS and trusted device methods
   - Prompt user for code appropriately

4. **Authentication Handshake Failed**
   - Verify SRP implementation is correct
   - Check encryption/decryption methods
   - Ensure proper key derivation
